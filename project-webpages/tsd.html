Train Station Database



The Purpose

The Train Station Database (TRD) is a mock database design and implementation for a train service to keep track of the status of each train, itâ€™s departure and arrival at different stations, the passengers or cargo on the train, the staff, and if a train is in for repairs and what those repairs are.

The Execution

The database is designed to be in 4th Normal Form and is written in postgreSQL. There is a basic python interface in order to query different information and the database uses a NoSQL form as well where information can be inserted and retrieved as a JSON file. This project was co developed by myself and Phillip Maphey
